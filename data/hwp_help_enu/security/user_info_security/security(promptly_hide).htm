Protect Selected Character
 

File > Document Security > Protect Private Info > Protect Selected Character

This feature changes selected block content within the active document into characters selected by the user and protects them with a password. The selected characters are protected by password encryption.

Steps

  1. Click File > New and enter data.
  2. Set the mobile phone number of Kim Hancom as a block.
  3. Click File > Document Security > Protect Private Info > Protect Selected Character.
  4. From the Set Protection Password dialog box, enter a password made up of 5 characters or more in Set Protection Password.
  5. In Confirm Password, type the same password again to record the correct password.
  6. Click Set. The password is applied to the document, and the mobile number that was set as a block is replaced with ***.
  7. Set a different section as a block and click Protect Selected Character to turn the selected characters into ***.
  8. Resting the pointer over a section that has been changed to *** alters the pointer into a lock, indicating that the section cannot be edited.

More Information

Set Protection Password Dialog Box

Set Protection Password

Type in a password made up of 5 characters or more.

Confirm Password

Type the password you typed in Document Password again.

Note

Password Entry Precautions

  • Typed Password Cannot Be Checked

  • If you type in a password, the characters are displayed as ***** to provide an additional layer of security. Only the person who types the password can remember the password.

  • Character Limit for Password

  • The protection password can range from 5 characters to 44 characters in length. The limit is not counted by the number of Korean or English letters but the number of keys pressed on the keyboard.

  • Keyboard Classification

  • For the protection password, the keys pressed on the keyboard are remembered, regardless of the type of the current keyboard. So you can mix English letters, numbers, and SPACEBAR on the keyboard.

  • Password Decoding Impossible

  • If you forget the protection password of a document, no should be able to modify, remove, or decode the user-defined password.

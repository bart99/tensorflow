Script Macro Security Settings: Security Level
 

View > Macros > Security Settings

You can set the level of script macro security from the Security Level tab of the Script Macro Security Settings dialog box.

Steps

To run macros at Security Level > Normal

  1. Click View > Task Pane > Script.
  2. From the Script task pane, create a macro that contains object-creation syntax.

    As an example, add a macro that runs Hancom Cell in the script edit window as shown below:

    function OnDocument_New()
    {
    //todo :
    var objname = "HNexcel.Application";
    var obj = new ActiveXObject(objname);
    obj.Visible =1;
    var msg = XHwpMessageBox;
    msg.String = objname + " is visible?";
    msg.DoModal();

    }

  3. View > Macros > Security Settings
  4. From the Script Macro Security Settings dialog box, select the Security Level tab.
  5. Set the security level to Normal and then click Set.
  6. From the Script task pane, click on the Run Script Macros icon .
  7. When the Script Macro Security Warning message displays, click Register and Execute.
  8. The macro that executes Hancom Cell is run, and the corresponding object is registered to the registry.

   You can check the list of registered objects in the Registered Objects tab of the Script Macro Security Settings dialog box.

More Information

Security Level Tab of Script Macro Security Settings Dialog Box

Very High

Does not allow macros to be run.

You will view a Script Macro Security Warning message if you run a script macro after setting Security Level to Very High. This message will alert you that the security level is set to Very High and that the macro cannot be run. To run a macro, you should set security level as Normal or Low.

High

Prompts the user for confirmation before running a script macro that contains objection-creation syntax, even if the object has already been registered.

If you run a script macro after setting Security Level to High, the macro will be handled as follows:

  • When running a macro without object creation parsing

  • A macro is run without asking whether to run it or not.

  • When running a macro with object creation parsing
    1. When an object not registered by the user is included

      A Script Macro Security Warning message appears, alerting you that the security level is set to High and the macro cannot be run because it contains an unregistered object. To run a macro, you should set security level as Normal or Low.

    2. When an object registered by the user is included

      A Script Macro Security Warning message appears, alerting you that the macro contains a registered object and creating it could lead to runtime of a virus. The user must then decide to allow or block the object-creation macro.

      Object creation macros may contain a virus or other security threats and should be disabled before use.

      You can check the list of registered objects in the Registered Objects tab of the Script Macro Security Settings dialog box.

      • Run Button

      • Run a macro without registering objects.

      • Block Button

      • Disables all macros.

Normal

Selectively prompts for user confirmation, depending on whether the script macro contains object-creation syntax and on whether the macro contains a registered object.

If you run a script macro after setting Security Level to Normal, the macro will be handled as follows:

  • When running a macro without object creation parsing

  • A macro is run without asking whether to run it or not.

  • When running a macro with object creation parsing
    1. When an object not registered by the user is included

      A Script Macro Security Warning message is shown, alerting you that the macro contains an unregistered object and creating it could lead to runtime of a virus. The user must then decide to allow or block the object-creation macro or to allow it to run after registering it.

      Object creation macros may contain a virus or other security threats and should be disabled before use.

      • Run Button

      • Run a macro without registering objects.

      • Register and Execute Button

      • Runs the macro without registering the object in the registry.

        You can check the list of registered objects in the Registered Objects tab of the Script Macro Security Settings dialog box.

      • Block Button

      • Disables all macros.

    2. When an object registered by the user is included

      The macro is run immediately and without displaying a Script Macro Security Warning message.

      You can check the list of registered objects in the Registered Objects tab of the Script Macro Security Settings dialog box.

Low (Not Recommended)

Allows all script macros to be run. Macros are run without prompting for your permission.

Selecting this setting exposes the system to risk of infection from virus-containing macros. Only use this setting if you have suitable protection set in place.

Note

Difference from Security Level

The macro may or may not be run, depending on the security level when a script macro containing syntax to create an object that has been registered by the user.

  • Security Level is set to Normal

  • Object-creation macros are allowed to run without displaying a Script Macro Security Warning message.

  • Security Level is set to High

  • When a script macro that contains object-creation syntax is run, a Script Macro Security Warning message prompts for your confirmation even if the object is a registered one.

Caution

Recommendation: Block Object-Creation Macros

Object creation macros may contain a virus or other security threats and should be disabled before use.

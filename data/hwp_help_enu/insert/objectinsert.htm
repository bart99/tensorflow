OLE
 

Insert > OLE

OLE objects are creations by applications for Windows. In programs like Hwp that are designed for creating documents, documents are objects. In programs designed for creating music, music files are objects. Similarly, objects can be characters, pictures (still pictures, movies), sound, charts, equations, tables, and others.

OLE (Object Linking & Embedding) refers to using an organic mixture of various data created by various applications in Windows, instead of using such data independently.

One example is embedding graphs, equations, pictures, and even sound data in Hwp documents. Another example is creating instructions for music data of a music program in a document editing program and then embedding the instructions in the music data.

The use of such independent data in one application is called OLE (Object Linking & Embedding).

Object Linking & Embedding can create efficient documents by using Hwp for document editing and other applications for various purposes.

Steps

To apply new OLE

  1. Click Insert > OLE.
  2. In the Insert Object dialog box, select New.
  3. From the Object Type list, select Paint Picture and click OK.
  4. When the Hwp edit window changes to the Paint program, draw a new picture.
  5. When you finish drawing a picture, click File > End Object Editing or click the outside of the Paint area to exit Paint.
  6. The picture will be inserted at the pointer in the document window.

To apply existing OLE

You can create various objects in advance and embed them in a document when needed.

  1. Click Insert > OLE.
  2. In the Insert Object dialog box, click Create from File. The object type list disappears and a box appears so you can type file name.
  3. Type the path and name of a Painter picture in the File box or select the picture file by clicking Browse.
  4. Select a picture file of Paint and click OK.
  5. The picture will be inserted at the pointer in the document window. Like this, a file object from another application gets embedded in the body text without starting the application. This is different from creating and embedding an object at the same time.

Note

Comparison of Object Linking and Object Embedding

Object Embedding is much easier to use when inserting an object in a document. If you use this function, however, codes about the object get embedded in the document as well and file size increases accordingly.

Once you use Object Linking, file size does not increase because objects exist as independent files. When you modify an object in another application, the change automatically applies without you applying it during document editing. The use of Object Linking is recommended.

  • Object Linking

  • When two programs share an object and the object which is linked to one of the programs changes, the object will have the same change in the other program.

  • Object Embedding

  • Changes only apply to the active program without affecting the original object.

Show as Icon

If you select Show as Icon, you can represent visible objects with picture files as they look. However, you cannot insert sound, videos, or special document files directly into the document. You have to display the objects with the icon of the application that can handle them, and connect the icon to the objects. You can check objects displayed as icons by double-clicking the icons.

Object Linking

When embedding an existing object, you can choose to link it or not. Selecting Create from File and Link to insert an object, you can edit it by clicking Home > OLE Connection.

Change Object Properties

Linked or embedded objects inserted in a document are treated as normal objects like pictures, tables, and drawings. So you can modify their size, anchor, and caption by clicking Home > Object Properties.

Insert Sound

You can insert sound objects by clicking Insert > Illustrations > Sound.

Caution

Appropriate Application Required

Object Linking & Embedding enables you to use data of different applications. You can also use data from another application without changing the data or embed the result in another application by using this function. You can play or modify the embedded object only if you have an application that can create or view the object. This reduces compatibility in some cases.

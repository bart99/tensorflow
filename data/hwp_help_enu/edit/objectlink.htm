OLE Connection
 

Home > OLE Connection

OLE: Object Linking & Embedding is one of the best functions provided by Windows. When two programs share material and an object linked to one of the programs changes while Object Linking is in use, the object in the other program changes as well. When Object Embedding is in use, only the active program changes without the original object getting affected.

Objects inserted through Object Linking contain information about the application in which they were created. Use Home > OLE Connection to modify objects inserted in a Hwp document and information about the applications in which they were created.

Steps

  1. Select a linked object and click Home > OLE Connection. Home > OLE Connection is only available when there is a linked object.
  2. In the Link dialog box, select an object to modify.
  3. Select Automatic for Update.
  4. Click Update Now. If the object is not the same as the active one, the object in the document is replaced with the original object.
  5. Click Close.

More Information

Link Dialog Box

Update

  • Automatic

  • Modify objects automatically whenever and wherever they change. Automatic update is used in general.

  • Manual

  • After objects change, apply the changes manually.

Update Now Button

If the object is not the same as the active one, the object in the document is replaced with the original object.

Open Source Button

If you click Open Source, the application that manages the object starts running and reads the file that contains the linked object. If you modify the object and exit the program, the change gets applied to the document.

Modify Source Button

If you click Modify Source, the Modify Source dialog box appears and you can select a new object. Select a new object that will replace the active object.

Break Link Button

It breaks the object link. If you break the link, changes of the original document will no longer apply to the active document.
